# PRD → 垂直切片 (Tracer Bullets)

基于 PRD V2.0 (`doc/prd1.md`) 拆解。

> **切片原则**：每个切片是贯穿所有层（DB → API → UI → 业务逻辑）的完整路径，完成后可独立演示。
> 当前项目已有前端脚手架 + 后端脚手架，不重复基础工程。

## S1: 句子库 + 核心训练交互

**类型**: HITL — 需确认句子数据格式和交互细节
**前置**: 无
**估时**: 2天

### 垂直范围
- **DB**: sentences 表 DDL + 种子数据 20 句
- **API**: `GET /api/train/sentence?mode=normal` — 获取单个句子
- **UI**: PAGE 1（入口页）→ PAGE 2（训练页）
- **逻辑**: 两态切换（黑字 ↔ 灰字+删除线）、重置、计时器、空提交检测

### 验收
- [ ] 进入 PAGE 1 看到标题和按钮
- [ ] 点击"开始训练"进入 PAGE 2，显示一个句子
- [ ] 点击单个词，在黑字和灰字+删除线之间切换
- [ ] 再次点击同一个词，恢复黑字
- [ ] 重置按钮一键恢复所有词为黑色
- [ ] 计时器正常走动
- [ ] 全灰点击提交 → 提示"请至少保留一个核心词"
- [ ] 全黑点击提交 → 正常通过

## S2: 评分引擎 + 对比展示

**类型**: AFK
**前置**: S1
**估时**: 1天

- [ ] 下方差异分析文本正确

## S3: 错题聚合回顾 + 针对性训练

**类型**: AFK
**前置**: S2
**估时**: 1.5天

### 垂直范围
- **API**: `GET /api/errors/review` — 按知识点聚合用户错题
- **API**: `GET /api/train/targeted?tag=xxx` — 获取同知识点句子
- **逻辑**: 错误分组 → 按标签聚合 → 取 Top 5
- **UI**: PAGE 3.5（错题回顾页）
- **流程**: 针对性训练闭环（PAGE 3.5 → PAGE 2 targeted 模式 → PAGE 3 → PAGE 3.5）

### 验收
- [ ] 训练提交后自动进入 PAGE 3.5
- [ ] 错题按知识点分组显示（定语从句/介词短语等）
- [ ] 每组显示错误次数
- [ ] 每条错题显示原文 + 正确做法
- [ ] "再来一组针对性训练"按钮 → 进入 PAGE 2（targeted 模式）
- [ ] 针对性训练完成后回到 PAGE 3.5，错题聚合更新
- [ ] 正常训练走 PAGE 3 → 3.5 → 4，针对性训练走 PAGE 3 → 3.5

## S4: LLM 智能反馈（含降级）

**类型**: HITL — 需确认 LLM 供应商、API Key、定价上限
**前置**: S2
**估时**: 1.5天

### 垂直范围
- **API**: `GET /api/feedback/{record_id}` — 获取结构化反馈
- **逻辑**: LLM 服务封装（组装 Prompt → 调用 API → 解析输出）、
  降级模板（10+ 知识点标签话术）、反馈缓存（同训练不重复调用）
- **DB**: feedback_cache 表读写
- **关联**: 训练记录提交后异步触发 LLM 调用

### 验收
- [ ] 训练提交后，后端自动调用 LLM 生成反馈
- [ ] LLM 返回结构化 JSON 并缓存
- [ ] 相同训练记录再次请求，命中缓存不重复调用
- [ ] LLM API 超时（>5s）→ 降级到规则模板，前端无感知
- [ ] LLM API 返回错误 → 降级到规则模板
- [ ] 降级模板覆盖 10+ 常见知识点标签

## S6: 微信 OAuth 用户系统

**类型**: HITL — 需要微信服务号 AppID + Secret 才能完成对接
**前置**: 无（可与 S1 并行）
**估时**: 1.5天

### 垂直范围
- **DB**: users 表
- **API**: `POST /api/user/login` — code2session 换 openid
- **API**: `GET /api/user/profile` — 用户信息
- **UI**: 微信 OAuth 静默授权跳转（前端）
- **逻辑**: localStorage openid 自动登录、渐进式信息收集（匿名→轻量→正式）

### 验收
- [ ] 用户首次访问，匿名体验
- [ ] 点击"领取诊断报告"触发微信 OAuth 静默授权
- [ ] 授权后获取 openid，写入 users 表
- [ ] openid 存 localStorage，后续访问自动登录
- [ ] 训练记录关联到 openid（不是匿名）

## S7: 微信支付闭环

**类型**: HITL — 需要微信商户号、API 密钥、回调域名等生产配置
**前置**: S6
**估时**: 2天

### 垂直范围
- **DB**: orders 表
- **API**: `POST /api/pay/create-order` — 统一下单
- **API**: `POST /api/pay/callback` — 微信异步回调
- **API**: `GET /api/pay/check-order` — 前端轮询
- **UI**: PAGE 6 调起 WeixinJSBridge 支付
- **逻辑**: 订单创建 → 调起支付 → 回调处理 → 用户权益更新（paid=1, 7天有效）
- **边界**: 掉单处理（轮询 60s + 手动刷新）、退款

### 验收
- [ ] PAGE 6 点击"开始7天训练"调起微信支付
- [ ] 支付成功 → 用户权益更新（paid=1）
- [ ] 支付失败 → 友好提示
- [ ] 微信异步回调 → 订单状态改为"已支付"
- [ ] 掉单 → 前端手动刷新可同步状态
- [ ] 退款 → 关闭用户权益

## S8: 7天训练闭环 + 数据看板

**类型**: HITL — 数据看板 UX 需确认
**前置**: S7, S3
**估时**: 2天

### 垂直范围
- **API**: `GET /api/train/daily` — 当日训练任务列表（已分配+已完成）
- **逻辑**: 每日分配 3 句、难度自适应（连续 3 次>90%升级、连续 2 次<60%降级）、防刷（每句每用户限一次）
- **UI**: 训练首页（付费用户 — 今日任务 + 完成状态 + 昨日 vs 今日对比 + 7天曲线）
- **UI**: "我的错题本"入口 + 独立页面
- **UI**: 每日首次打开横幅提醒（"你还有 N 个薄弱知识点未攻克"）

### 验收
- [ ] 付费用户登录后进入训练首页
- [ ] 显示今日 3 句训练任务，标注完成状态
- [ ] 昨日 vs 今日压缩指数对比
- [ ] 7 天趋势曲线
- [ ] 连续 3 次 >90% → 难度自动升级
- [ ] 连续 2 次 <60% → 难度降级
- [ ] 已完成句子不可重复训练
- [ ] "我的错题本"页面显示历史累计错题

## 建议开发顺序

```
S1 句子+训练   ─────┐
                    ├── 核心链路
S6 用户系统    ─────┘
                    │
S9 句子管理    ─────┘（运营并行）
```

### 第 2 批（依赖 S1）
```
S2 评分对比   →   S3 错题回顾
                →   S4 LLM反馈
```

### 第 3 批（依赖 S2/S3/S4/S6）
```
S5 转化漏斗   →   S7 支付闭环   →   S8 7天训练
```

### 第 4 批
```
S10 部署收尾
```

## 快速启动建议

1. **S1 句子库 + 核心训练** — 已完成前端 80%、后端 60%，联调即可
2. **S9 句子管理** — 运营同事可立即开始标注
3. **S6 用户系统** — 可先 mock OAuth，业务逻辑先行
